# Continual Agent Learning

# 1. 自治域

## 1.1 现有 Agent 的缺陷
很多现有 agent 系统，虽然表面上在“用工具”，但本质上仍然是在**纯语言空间**里工作：

模型输出文本 -> 被 parser 解析成 tool call -> 工具返回文本 -> 文本再被转化为上下文

ExPA 的思路：

> 现有方法把**语言推理**和**环境交互**混在同一个语言动作空间里，导致模型同时承担 reasoning 和 control 两种职责

ExPA 的核心改进是把环境交互从语言里“拿出来”，让模型可以通过显式 routing action 进入外部环境，并在环境内部只选择环境特定动作

**agent 按环境或能力单元进行模块化组织更合理高效**

## 1.2 为什么这个问题在 continual setting 下更严重
如果系统是静态的，扁平动作空间虽然笨重，但勉强还能工作  
但在 continual agent learning 里，系统会不断变化：
- 新工具加入
- 旧工具升级
- 新环境加入
- 旧环境内部动作变多
- 跨工具组合模式变复杂

这时如果还维持一个统一扁平动作空间，那么每加入一个新能力，都相当于在**全局策略**上做一次扰动  
Structured Agent slides 已经把这个问题概括成 **tool-space interference**，它的表现包括：
- performance degradation
- context overloading
- semantic collisions

所以我们需要一个新的基本抽象，使得系统增长时：
1. 新能力能被**局部接入**
2. 旧能力不会被**全局污染**
3. 跨能力协作仍然能被**显式建模**

这里我借用计算机网络中的概念，称之为自治域（Autonomous System）

# 2. 自治域的直观含义
一个自治域不是简单的“一个工具名”，而是一个**局部可控的交互子系统**

## 2.1 AS有自己的局部动作规则
例如 calculator 域里的动作不是自然语言，而是：
- 数字按钮
- 运算符
- 等号

table 域里的动作可能是：
- select column
- sort
- sum
- plot

这些动作的语义和 browser 或 filesystem 完全不同

### 第二，它有自己的局部状态与反馈

例如：

- calculator 有内部表达式状态
    
- filesystem 有文件树状态
    
- browser 有页面 DOM / navigation 状态
    
- document agent 有文档片段与结构状态
    

### 第三，它允许局部封闭决策

一旦进入该域，模型不需要再面对全局所有动作，只需要在该域允许的动作上决策。

这正是 Structured Agent slides 里“reduction of action space”的核心直觉：  
对于结构化数据和结构化任务，更小、更局部、更数据结构感知的动作空间更适合决策，也更利于 continual/lifelong learning。

---

# 3. 自治域的形式化定义

现在给出最小且统一的定义。

我们定义在阶段 (k) 时，系统包含一组自治域：

[  
\mathcal{D}^{(k)}={D_0,D_1,\dots,D_{K_k}}.  
]

其中：

- (D_0) 表示默认的**语言域**
    
- (D_i) 表示第 (i) 个外部自治域，例如 calculator、table、filesystem、browser
    

每个自治域 (D_i) 定义为四元组：

[  
D_i = (\mathcal{O}_i,\mathcal{A}_i^{(k)},P_i,\beta_i).  
]

这里：

- (\mathcal{O}_i)：该域的局部观测空间
    
- (\mathcal{A}_i^{(k)})：阶段 (k) 时该域的局部动作空间
    
- (P_i)：该域的局部状态转移机制
    
- (\beta_i)：退出机制，决定何时离开该域
    

这个定义有意保持克制。  
它只抓住最核心的事情：**一个域是一套局部观测-动作-转移-退出规则。**

---

# 4. 为什么这个定义是必要的

这个定义解决了三个以前混在一起的问题。

## 4.1 它把“动作语义”局部化了

在扁平系统中，所有动作都在一个大 softmax 里竞争。  
在自治域系统中，动作先按域分组：

[  
\mathcal{A}^{(k)} = \mathcal{A}_0 \cup \mathcal{A}_1^{(k)} \cup \cdots \cup \mathcal{A}_{K_k}^{(k)}.  
]

于是新增一个动作，不再是对整个全局动作空间的污染，而是对某个域的局部扩展。

---

## 4.2 它把“能力增长”变成了结构化增长

在 continual setting 里，系统增长有两种最自然的方式：

### 新增一个域

[  
\mathcal{D}^{(k)} = \mathcal{D}^{(k-1)} \cup {D_{\text{new}}}  
]

### 扩展某个域的局部动作

[  
\mathcal{A}_i^{(k)} = \mathcal{A}_i^{(k-1)} \cup \Delta \mathcal{A}_i^{(k)}  
]

这比“往全局工具列表里再加一个名字”更有结构，也更符合 continual learning 的需要。

---

## 4.3 它把“全局策略”分解成路由与局部控制

这一步最重要。

如果系统由多个自治域组成，那么 agent 的决策天然分成两部分：

1. **我现在该进入哪个域？**
    
2. **进入这个域之后，我该做什么？**
    

于是全局决策不再是一个黑盒，而是分解成：

- 域间路由
    
- 域内控制
    

这正是你后面三层问题定义的基础。

---

# 5. 全局状态与策略的统一定义

为了统一后面的公式，我们定义在时间步 (t) 的可观测交互状态为：

[  
x_t = (h_t, e_t, o_t),  
]

其中：

- (h_t)：到当前为止的历史表示，可以是 token history，也可以包含 memory readout
    
- (e_t \in {0,1,\dots,K_k})：当前激活的自治域
    
- (o_t \in \mathcal{O}_{e_t})：当前域的局部观测
    

如果你想与 ExPA 对齐，也可以把潜在环境状态写成更完整的 POMDP 形式：

[  
s_t=(h_t,e_t,z_t),  
]

其中 (z_t) 是不可完全观测的外部环境隐状态。ExPA 的 formulation 就是这一写法。

---

## 5.1 策略分解

我们把系统策略写成两级：

### 域间路由策略

[  
\pi_{\mathrm{route}}^{(k)}(d' \mid h_t,e_t)  
]

表示当前是否留在当前域，或切换到另一个域 (d')。

### 域内控制策略

[  
\pi_{i}^{(k)}(a \mid h_t,o_t,e_t=i),  
\qquad a\in \mathcal{A}_i^{(k)}.  
]

于是整体策略可以理解为：

[  
\Pi^{(k)}=\Big(\pi_{\mathrm{route}}^{(k)},{\pi_i^{(k)}}_{i=0}^{K_k}\Big).  
]

这里的核心思想很简单：

> **全局策略不直接对整个系统的所有动作做一次性决策，而是先决定“在哪个域行动”，再决定“在域内如何行动”。**

---

# 6. 这样定义之后，Continual Agent Learning 到底在学什么

现在就可以自然引出 continual learning 的对象。

我们不是把 continual learning 看成“模型不断微调”，而是看成系统能力空间的持续扩展。

在阶段 (k)，系统可能发生两类基本变化：

## 6.1 域内增长

某个既有自治域 (D_i) 增加了新的局部动作：

[  
\mathcal{A}_i^{(k)}=\mathcal{A}_i^{(k-1)} \cup \Delta\mathcal{A}_i^{(k)}.  
]

例如 table 域从原先只有 `select`、`filter`，扩展到 `sort`、`plot`、`pivot`。

这对应你要讲的第一层问题：

> **在一个固定自治域内部，局部动作空间持续扩展时，如何学新动作而不忘旧动作？**

---

## 6.2 域间增长

系统加入一个新的自治域：

[  
\mathcal{D}^{(k)}=\mathcal{D}^{(k-1)}\cup{D_{\text{new}}}.  
]

例如原先只有 calculator 与 table，现在加入 filesystem 或 browser。

这对应第二层问题：

> **自治域集合增长时，如何学会新的路由与组合，而不扰乱旧的域间边界？**

---

# 7. 为什么这种分解对 continual learning 更合理

现在可以把最核心的逻辑讲出来。

在传统 continual learning 里，我们常常按“任务”划分阶段。  
但对 agent 来说，真正自然的增量对象不是 task label，而是：

- 新域
    
- 新局部动作
    
- 新组合模式
    

因为 agent 的能力本身就是**模块化接入**的。

因此，自治域的意义在于：

## 7.1 它给 continual learning 提供了更自然的基本单位

不是“任务 1、任务 2、任务 3”，而是：

- 域 (D_i) 的扩展
    
- 域集 (\mathcal{D}) 的扩展
    

## 7.2 它让抗遗忘从“全局保护”变成“局部保护”

如果不分域，新增一个能力会扰动整个策略。  
如果分域，则新增能力主要影响：

- 某个域的局部策略 (\pi_i)
    
- 或顶层路由策略 (\pi_{\mathrm{route}})
    

这就把更新局部化了。

## 7.3 它让跨域组合成为显式问题

当多个自治域共存时，agent 任务往往不是单域完成的，而是：

- browser (\rightarrow) table
    
- filesystem (\rightarrow) document
    
- browser (\rightarrow) document (\rightarrow) calculator
    

这种组合不再是“prompt 里碰巧出现的一段模式”，而是**域间路由和局部控制共同形成的结果**。

---

# 8. 用一个统一目标把三层问题串起来

现在给出最核心的一条目标函数。

在阶段 (k)，系统面对任务分布 (\mathcal{T}^{(k)})，目标不是单纯最大化当前任务收益，而是同时兼顾：

1. 当前适应
    
2. 旧能力保持
    
3. 跨域干扰抑制
    

最一般地，可以写成：

## [  
\max_{\Pi^{(k)}}  
\quad  
\underbrace{  
J_{\text{new}}^{(k)}(\Pi^{(k)})  
}_{\text{当前阶段适应}}

## \lambda_{\text{forget}}  
\underbrace{  
\mathcal{F}^{(k)}  
}_{\text{遗忘}}

\lambda_{\text{int}}  
\underbrace{  
\mathcal{I}^{(k)}  
}_{\text{结构干扰}}.  
]

这里：

- (J_{\text{new}}^{(k)})：当前阶段任务上的期望回报
    
- (\mathcal{F}^{(k)})：对历史能力的退化
    
- (\mathcal{I}^{(k)})：新增域或新增动作带来的结构性干扰
    

这条式子就是整个框架的总目标。

下面三层，其实只是这条总目标在不同层次上的具体化。

---

# 9. 第一层：域内持续学习

先看一个固定自治域 (D_i)。

在阶段 (k)，它的局部动作空间扩展为：

[  
\mathcal{A}_i^{(k)}=\mathcal{A}_i^{(k-1)}\cup \Delta \mathcal{A}_i^{(k)}.  
]

这时我们希望学会新动作，同时尽量不破坏旧动作的使用能力。  
因此域内目标写成：

## [  
\max_{\pi_i^{(k)}}  
\quad  
J_{i,\text{new}}^{(k)}

\lambda_{\text{intra}}  
\mathcal{F}_{i,\text{old}}^{(k)}.  
]

其中：

- (J_{i,\text{new}}^{(k)})：该域在新动作加入后的当前任务回报
    
- (\mathcal{F}_{i,\text{old}}^{(k)})：旧动作行为的偏移程度
    

如果你想具体一些，可以把保持项写成旧域上下文上的策略偏移：

# [  
\mathcal{F}_{i,\text{old}}^{(k)}

\mathbb{E}_{h\sim \mathcal{B}_{i}^{(<k)}}  
\Big[  
D_{\mathrm{KL}}  
\big(  
\bar{\pi}_{i}^{(k-1)}(\cdot\mid h)  
\parallel  
\bar{\pi}_{i}^{(k)}(\cdot\mid h)  
\big)  
\Big].  
]

这里 (\bar{\pi}_i) 表示**只在旧动作子集上重新归一化后的限制分布**。  
这样做的意义是：

> 我们允许新动作得到概率质量，但不希望旧动作之间原本学到的相对偏好被破坏。

这就是第一层最核心的公式。

---

# 10. 第二层：域间持续学习

接着看系统层面的域增长。

在阶段 (k)，加入新域：

[  
\mathcal{D}^{(k)}=\mathcal{D}^{(k-1)}\cup {D_{\text{new}}}.  
]

这时问题不再只是“域内动作变多”，而是：

- 什么时候该进入新域
    
- 什么时候不该误入新域
    
- 新域加入后会不会扰乱旧域的路由边界
    

所以第二层主要约束的是**路由策略**。

写成目标：

## [  
\max_{\pi_{\mathrm{route}}^{(k)}}  
\quad  
J_{\mathrm{route,new}}^{(k)}

## \lambda_{\mathrm{route}}  
\mathcal{F}_{\mathrm{route}}^{(k)}

\gamma_{\mathrm{col}}  
\mathcal{C}_{\mathrm{collision}}^{(k)}.  
]

其中：

- (J_{\mathrm{route,new}}^{(k)})：新域相关任务上的收益
    
- (\mathcal{F}_{\mathrm{route}}^{(k)})：旧域之间路由边界的漂移
    
- (\mathcal{C}_{\mathrm{collision}}^{(k)})：在旧任务上下文中误路由到新域的概率
    

如果写得更具体一点，可以把 collision 项理解为：

# [  
\mathcal{C}_{\mathrm{collision}}^{(k)}

\mathbb{E}_{h\sim \mathcal{B}_{\mathrm{old}}}  
\big[  
\pi_{\mathrm{route}}^{(k)}(D_{\text{new}} \mid h)  
\big].  
]

它的意义非常直观：

> 在旧上下文里，别把新域错当成正确选择。

这正对应 slides 里说的 semantic collisions / tool-space interference。

---

# 11. 第三层：系统级持续学习

最后才来到整个系统。

阶段 (k) 的系统定义为：

[  
\mathcal{M}^{(k)}=  
\big(  
\mathcal{D}^{(k)},  
\Pi^{(k)},  
\mathcal{T}^{(k)},  
\mathcal{B}_{\mathrm{mem}}^{(k)}  
\big).  
]

这里：

- (\mathcal{D}^{(k)})：当前自治域集合
    
- (\Pi^{(k)})：整体策略系统
    
- (\mathcal{T}^{(k)})：当前任务分布
    
- (\mathcal{B}_{\mathrm{mem}}^{(k)})：记忆模块
    

Structured Agent slides 提出 memory 应分成 token、parametric / structured、latent 三层，而不是只把所有历史都堆进上下文。  
所以我们把 memory 写成：

# [  
\mathcal{B}_{\mathrm{mem}}^{(k)}

\mathcal{B}_{\mathrm{tok}}^{(k)}  
\cup  
\mathcal{B}_{\mathrm{obj}}^{(k)}  
\cup  
\mathcal{B}_{\mathrm{lat}}^{(k)}.  
]

于是系统级目标写成：

## [  
\max_{\Pi^{(k)}}  
\quad  
J^{(k)}(\Pi^{(k)})

\lambda_{\mathrm{mem}}  
\mathcal{F}_{\mathrm{mem}}^{(k)}.  
]

其中：

# [  
\mathcal{F}_{\mathrm{mem}}^{(k)}

\mathbb{E}_{(h,e,m)\sim \mathcal{B}_{\mathrm{mem}}^{(<k)}}  
\Big[  
D_{\mathrm{KL}}  
\big(  
\pi^{(k-1)}(\cdot\mid h,e,m)  
\parallel  
\pi^{(k)}(\cdot\mid h,e,m)  
\big)  
\Big].  
]

这里的含义是：

> 旧经验不一定都要以 raw token 的形式重放；它们可以被压缩成结构化记忆或 latent 先验，再用于约束新策略不要在历史能力上漂移过远。

这一步就是把 slides 里的 hierarchical memory 和 continual learning 正式接起来。

---

# 12. 三层之间的关系

你在介绍时，最好把三层关系讲成**递进**，而不是三个分散模块。

## 第一层：域内

固定一个自治域，研究局部动作空间扩展。  
核心问题是：

> **局部能力如何持续增长而不忘旧动作？**

## 第二层：域间

自治域集合增长，研究路由与组合。  
核心问题是：

> **新域加入后，如何保持旧域边界稳定并避免语义碰撞？**

## 第三层：系统级

任务、域、动作、记忆共同演化。  
核心问题是：

> **整个系统如何通过结构化记忆与局部更新实现长期进化？**

所以最简洁的一句话是：

> **自治域是把 Continual Agent Learning 从“一个统一大模型的持续微调问题”重写成“一个模块化交互系统的持续扩展问题”的基本抽象。**

---

# 13. 你给他们讲时最适合的 opening

你可以直接这样说：

> 我想先从一个最小抽象开始：自治域。  
> 引入自治域的动机，是因为在 continual agent learning 里，系统能力不是一次性给定的，而是在不断增长。如果我们仍然把所有工具和交互都放在统一扁平动作空间里，那么每次新增能力都会扰动整个策略，导致 interference 和 forgetting。  
> 所以我更倾向于把 agent 系统看成由多个自治域组成：每个域有自己的局部观测、局部动作、局部转移和退出机制。这样全局决策就自然分成“域间路由”和“域内控制”。在这个基础上，continual learning 也就自然分成三个层次：域内动作扩展、域间路由稳定，以及系统级的长期演化。

这段就很适合作为你后续公式部分的总引子。
