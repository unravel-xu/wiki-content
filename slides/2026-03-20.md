# Continual Agent Learning
[Lifelong Learning of Large Language Model based Agents: A Roadmap](https://github.com/qianlima-lab/awesome-lifelong-llm-agent)（TPAMI 2026）

# 自治域
类似计算机网络中的概念，这里我简单定义一个自治域是一个局部封闭的交互环境，它有自己的观测、动作和状态转移。当 Agent 进入这个域，决策空间就被限制在该域的局部动作集合中

形式上，一个域 $D$ 定义为：
$$
D = (\mathcal{O}, \mathcal{A}, P, \beta)
$$
其中：
- $\mathcal{O}$：局部观测环境
- $\mathcal{A}$：局部动作空间
- $P$：域内状态转移
- $\beta$：退出条件

> ExPA 的关键在于进入环境后只在该环境允许的动作上决策

所以一个自治域就是 ExPA 里的一个外部环境的抽象

# 基本设定：基于 ExPA 框架的符号对齐

首先，我们复用 ExPA 论文（第 3 节）中的 POMDP 设定：
*   **全局状态 (Global State):** $s_t = (h_t, e_t, z_t)$，其中 $h_t$ 是 token 历史，$e_t \in \{0, 1, \dots, K\}$ 是当前激活的自治域（$e_t=0$ 为语言域，其他为外部环境域），$z_t$ 是外部环境的隐状态
*   **动作空间 (Action Space):** 包含语言词表 $\mathcal{V}$ 和扩展动作空间 $\mathcal{E}$。其中 $\mathcal{E} = \bigcup_{i=1}^K \{g_i\} \cup \mathcal{E}_i$。$g_i$ 是进入自治域 $i$ 的**路由动作 (routing action)**，$\mathcal{E}_i$ 是自治域 $i$ 内部的**局部动作集 (environment-specific actions)**
*   **策略 (Policy):** 统一表示为 $\pi_\theta(a_t \mid h_t, e_t)$

在 Continual Learning 设定下，我们假设系统经历离散的学习阶段（Tasks/Stages） $k = 1, 2, \dots, N$

# 第一层：域内持续学习 (Intra-domain Continual Learning)
局部动作空间的持续扩展
**动机：** 在单一自治域 $i$ 内（此时 $e_t = i$），工具的能力是在不断升级的（例如 Table 域从只支持 `search` 扩展到支持 `sort` 和 `plot`）

**形式化定义：**
在阶段 $k$，自治域 $i$ 的局部动作空间发生扩展：
$$ \mathcal{E}_i^{(k)} = \mathcal{E}_i^{(k-1)} \cup \Delta \mathcal{E}_i^{(k)} $$
对于域内决策策略 $\pi_{\text{local}}(a \mid h_t, e_t=i; \theta^{(k)})$，模型需要学会使用新动作集合 $\Delta \mathcal{E}_i^{(k)}$，同时不丧失对旧动作集 $\mathcal{E}_i^{(k-1)}$ 的精准控制。

**优化目标：**
设该自治域在阶段 $k$ 的任务奖励为 $R^{(k)}$。我们在最大化新任务奖励的同时，对旧动作的策略分布施加正则化约束（例如 KL 散度），防止灾难性遗忘：
$$ \max_{\theta} \mathbb{E}_{\tau \sim \pi_\theta, \mathcal{E}_i^{(k)}} \left[ \sum R^{(k)}(s_t, a_t) \right] - \lambda_{\text{intra}} \mathbb{E}_{h_t} \left[ D_{\text{KL}} \Big( \pi_{\text{old}}(\cdot \mid h_t, e_t=i) \parallel \pi_{\theta}(\cdot \mid h_t, e_t=i) \Big)_{\mathcal{E}_i^{(k-1)}} \right] $$
*(注：这里下标 $\mathcal{E}_i^{(k-1)}$ 表示 KL 散度仅在旧动作子集上计算并归一化，允许新动作概率自然增长，但不破坏旧动作之间的相对偏好。)*

💡 **与他们工作的结合点 (Pitch)：** ExPA 在 Section 4.1 中强调了对新动作的 Initialization（用自然语言 token 的 embedding 初始化新动作）。但在 CL 阶段，单纯的初始化不足以防止旧权重被 RL 梯度破坏。这一层的公式补充了动作空间动态增长时的参数保护机制。

### 🟡 第二层：域间持续学习 (Inter-domain Continual Learning)
**—— 自治域增长下的路由与组合防冲突 (Routing Stability & Semantic Collision Avoidance)**

**动机：** 这是对应 Slides 第 7 页 "Tool-space interference / Semantic Collisions" 的核心。当系统引入全新的工具/环境时，如何保证不扰乱原有的工具调用逻辑？

**形式化定义：**
在阶段 $k$，系统引入了一个全新的自治域 $D_{\text{new}}$。此时外部环境集合扩展：
$$ \mathcal{D}^{(k)} = \mathcal{D}^{(k-1)} \cup \{D_{\text{new}}\} $$
对应到 ExPA 的动作空间，即增加了一个新的路由动作 $g_{\text{new}}$：
$$ \mathcal{G}^{(k)} = \mathcal{G}^{(k-1)} \cup \{g_{\text{new}}\} \quad (\text{其中 } \mathcal{G} \text{ 是所有 } g_i \text{ 的集合}) $$

在语言域（$e_t = 0$）中，策略承担着**域间路由 (Routing Policy)** 的功能：
$$ \pi_{\text{route}}(a \mid h_t, e_t=0; \theta), \quad a \in \mathcal{V} \cup \mathcal{G}^{(k)} $$

**面临的挑战（Semantic Collision）：**
当引入 $g_{\text{new}}$（如 `<Python>`）时，模型可能会在旧任务（如简单的加减法）中错误地将高概率分配给 $g_{\text{new}}$，而不是原本正确的 $g_{\text{calc}}$（`<Calculator>`）。

**优化目标：**
我们需要一个 **Routing-Preservation** 机制。对于属于旧任务分布 $h_t \sim \mathcal{T}^{(<k)}$ 的上下文，约束新策略的路由分布：
$$ \mathcal{L}_{\text{route\_collision}} = \mathbb{E}_{h_t \sim \mathcal{T}^{(<k)}} \left[ D_{\text{KL}} \left( \pi_{\text{old}}(g \mid h_t, e_t=0) \parallel \pi_\theta(g \mid h_t, e_t=0) \right)_{\mathcal{G}^{(k-1)}} + \gamma \cdot \pi_\theta(g_{\text{new}} \mid h_t, e_t=0) \right] $$
*(公式解释：第一项确保旧工具之间的选择偏好不发生改变；第二项惩罚在旧任务上下文中错误调用新工具的概率 $\gamma$ 为惩罚系数。)*

💡 **与他们工作的结合点 (Pitch)：** ExPA 提出了 Counterfactual Rollouts（CPO，Section 4.2）。在这层，我们可以提出 **Continual CPO**：当引入新工具时，我们人为构造 Counterfactual trajectory，强迫模型在旧任务中尝试走 $g_{\text{new}}$ 的分支，如果发现 Reward 极低（即 Semantic Collision），则给予负向梯度，从而划清不同自治域之间的路由边界。

### 🔴 第三层：系统级持续学习 (System-level Continual Learning)
**—— 基于记忆机制的全局策略进化 (Global Evolution with Memory-augmented Structuring)**

**动机：** 当任务分布 $\mathcal{T}$、环境集合 $\mathcal{D}$、局部动作集 $\mathcal{E}$ 都在随时间 $k$ 演变时，我们需要一个系统级的统一目标。结合 Slides 第 8-14 页提出的 Dynamic Memory Handling（特别是 Latent/Soft memory）。

**形式化定义：**
整个 Agent 系统在阶段 $k$ 定义为：
$$ \mathcal{M}^{(k)} = \left( \mathcal{D}^{(k)}, \Pi^{(k)}, \mathcal{T}^{(k)}, \mathcal{B}_{\text{mem}}^{(k)} \right) $$
其中 $\mathcal{B}_{\text{mem}}$ 是系统的记忆模块（可以是 Token-level 的 Exemplars，或者是 Parametric/Latent 层的 Soft Memory）。

在阶段 $k$，我们需要在当前任务分布 $\mathcal{T}^{(k)}$ 上最大化优势函数（Advantage），同时依赖记忆模块限制对历史策略的偏离（Forgetting）。

**全局统一优化目标 (Global Objective)：**
结合 ExPA 的 CPO 目标函数（论文 Equation 1），我们将 CL 引入：
$$ \mathcal{J}_{\text{Continual-CPO}}(\theta^{(k)}) = \mathbb{E}_{s_0 \sim \mathcal{T}^{(k)}} \left[ \frac{1}{m} \sum_{i=1}^m U_i \big( \mathcal{T}(s_0); \theta^{(k)} \big) \right] - \lambda \mathcal{F}_{\text{memory}} \left( \theta^{(k)}, \theta^{(k-1)}, \mathcal{B}_{\text{mem}}^{(<k)} \right) $$

其中遗忘惩罚项 $\mathcal{F}_{\text{memory}}$ 定义为历史记忆上的性能差或分布散度：
$$ \mathcal{F}_{\text{memory}} = \mathbb{E}_{(h, e) \sim \mathcal{B}_{\text{mem}}^{(<k)}} \left[ D_{\text{KL}} \Big( \pi_{\theta^{(k-1)}}(\cdot \mid h, e) \parallel \pi_{\theta^{(k)}}(\cdot \mid h, e) \Big) \right] $$

💡 **与他们工作的结合点 (Pitch)：** 
这层的核心价值在于告诉合作者：**“你们 Slides 里讲的 Hierarchical Memory，正是解决系统级 CL 遗忘问题的钥匙。”**
*   如果不加 Memory，$\mathcal{F}_{\text{memory}}$ 只能计算当前任务数据，会导致灾难性遗忘。
*   通过结合 Slides 第 13 页的 "Concept token / Continuous concept space"，我们可以将旧工具的正确调用范式压缩为 Latent Memory $z_{\text{old\_tasks}}$ 注入到 $h_t$ 中，这样计算上述 KL 散度时计算成本极低，且能实现 Lifelong Agent
