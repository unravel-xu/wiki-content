# Unified Post-Training
打破 SFT 和 RL 的两阶段割裂，实现单阶段/统一后训练
如下我把 **Unified Post-Training** 拆成四个层次：沿着 **目标/梯度、数据、调度、优化层级** 这四个层次分类目前的工作

## 目标函数与梯度估计的统一
核心思想：SFT 和 RL 表面上一个是监督学习、一个是强化学习，但从策略优化角度看，它们都在更新同一个语言策略，只是采用了不同的数据分布、不同的 advantage 构造方式、不同的稳定化手段。因此，“统一”首先可以发生在 **梯度估计器** 或 **目标函数分解** 上

### Towards a Unified View of Large Language Model Post-Training (HPT / UPGE)

ICLR 2026 虚假引用被 Desk Reject

它推导出一个 **Unified Policy Gradient Estimator, UPGE**，把大范围的后训练方法都写成同一目标下的不同梯度估计形式，并拆成四个可替换组件：**stabilization mask、reference policy denominator、advantage estimate、likelihood gradient**。它的关键贡献不是单独发明一个新 loss，而是说明很多后训练算法其实共享一套更底层的梯度骨架。基于这个统一视角，作者又提出 **HPT**，在训练中动态切换不同训练信号

公式都抽象成了 $\nabla_\theta J = \mathbb{E}[ \text{Mask} \cdot \text{Advantage} \cdot \nabla_\theta \log \pi_\theta ]$
*   **如何统一**：在这个公式下，**SFT** 等价于 Advantage 永远为 1，且只在专家数据（Demonstrations）上 Mask 为 1（模型自己采样的轨迹 Mask 为 0）。而 **PPO** 则是 Advantage 为真实的 GAE，Mask 覆盖模型探索的轨迹。**DPO** 则相当于 Advantage 是隐式的（基于正负样本的 reward 差分计算）

### ORPO: Monolithic Preference Optimization without Reference Model
ACL 2024
ORPO 可以看作这一层里更“工程化”的统一方式。它不再把“先 SFT、再 preference optimization”拆成两个阶段，而是直接将偏好对齐（Alignment）直接融入 SFT 阶段。它不需要参考模型（Reference Model），也不需要单独的 RL 阶段，而是直接在传统的 SFT 交叉熵 Loss 后面加了一个基于赔率比（Odds Ratio）的惩罚项

ORPO 的 Loss 分为两部分：$L_{SFT} - \lambda L_{OR}$。其中 $L_{OR}$ 是对数赔率比（Log Odds Ratio）。赔率 $odds = \frac{P(y|x)}{1-P(y|x)}$

> SFT 只会告诉模型什么是好答案，但不一定会让它主动远离坏答案
> ORPO 在 SFT 目标之外，加了一个“好答案要比坏答案更占优”的约束
> 因此它把偏好对齐直接融进监督微调里，用一个单阶段目标同时完成“学习”和“纠偏”

alignment 目标本身可以被写进一个单阶段、单目标的训练式中

## 数据来源与策略分布的统一
核心思想：把问题从“训练目标不一样”转成“训练数据是从哪里来的不一样”。在过去训练范式里，SFT 和 RL 好像是两套不同的方法；但越来越多的新工作开始强调，真正关键的区别不一定在目标函数上，而在于模型训练数据的来源不同

SFT 主要依赖的是提前准备好的示范答案，也就是专家或者人工整理好的标准解法
RL 主要依赖的是模型自己当前能力下现场尝试出来的解题过程

所以真正的统一，不只是把两个损失函数写到一起，而是把“别人给的示范”和“模型自己试出来的过程”都放到同一个策略改进框架里
  
如果最后结果有好有坏，我们应该把鼓励和修正分配到哪些推理步骤上

### Large Language Model Post-Training: A Unified View of Off-Policy and On-Policy Learning
这篇综述最大的启发是，它不再按 SFT、DPO、RL 这些方法名字来分类，而是按“训练过程是谁提供的”来分类。一类是模型学习别人已经准备好的高质量示范，另一类是模型学习自己当前探索出来的过程。这样看之后，统一后训练的核心问题就变成了：模型到底是在向谁学习，以及外部示范和自主探索这两类过程如何协同

### On-Policy RL Meets Off-Policy Experts: Harmonizing Supervised Fine-Tuning and Reinforcement Learning via Dynamic Weighting
ICLR 2026

CHORD 明确用 **off-policy vs. on-policy** 的视角来重写 SFT 与 RL 的关系，把 SFT 不再看成一个独立阶段，而是看成 on-policy RL 过程中的 **动态加权辅助目标**。它同时设计了全局系数和 token-level 权重，让模型既能利用专家数据，又不至于被离线数据过度牵着走。这个视角非常重要，因为它说明：统一训练的核心不是“都叫 post-training”，而是 **不同来源轨迹如何共同塑造策略**

### HPT
HPT 其实也属于这一层，因为它从一开始就把后训练数据分成 **online rollouts** 和 **offline demonstrations** 两类，并把它们放入同一个统一策略梯度框架来分析。也就是说，HPT 同时横跨“梯度统一”和“数据统一”两层

模型用的是外部专家轨迹，还是当前策略采样出的轨迹，如何处理两类数据之间的分布冲突

## 训练调度与信用分配的统一
核心思想：它承认 SFT 和 RL 在训练信号性质上并不一样：SFT 提供密集、稳定、低方差的监督；RL 提供更开放、更灵活、但往往更稀疏的反馈。因此统一的关键，不再只是写一个总目标，而是设计一个 **controller / scheduler**，决定在每个 batch、每个样本，甚至每个 token 上，究竟应该更依赖哪种信号

### SRFT: A Single-Stage Method with Supervised and Reinforcement Fine-Tuning for Reasoning
ICLR 2026

SRFT 的核心观察是：从熵视角看，SFT 更倾向于引起粗粒度、全局性的策略分布偏移，而 RL 更像做细粒度、选择性的局部优化。在单阶段训练中，如果对于一个问题，模型生成的几条轨迹的困惑度（Perplexity/Entropy）很高，说明模型根本不知道怎么解，此时 RL 给的信号（哪怕是偶然猜对的）方差极大，极易导致崩溃。SRFT 根据高熵状态动态增大 SFT 权重，让模型“先学走”；如果熵很低（模型很自信），则依靠 RL 去微调细节，“再学跑”。它实际上做了一个基于认知状态的 Scheduler

### SuperRL: Reinforcement Learning with Supervision to Boost Language Model Reasoning
ICLR 2026 reject

SuperRL 针对的是 reasoning 中非常现实的问题：奖励稀疏。它的做法很直接：对于一个训练样本，先采样多条 rollout；只要有 rollout 获得非零奖励，就进行 policy-gradient 更新；如果所有 rollout 奖励都为 0，说明当前 RL 没有可用学习信号，就触发 fallback，回退到专家数据上的 SFT。它把统一训练写成一种实例级的“自适应切换机制”，本质上是在解决 credit assignment 失败时如何由监督信号接管

### CHORD
CHORD 也可以放在这一层，因为它不仅统一了数据来源，还进一步做了 dual-control dynamic weighting：先用全局系数控制从 imitation 向 exploration 的过渡，再用 token-wise 权重对专家 token 进行细粒度筛选。它引入的动态加权可能基于“策略差异度”（如 $\pi_\theta$ 和 $\pi_{expert}$ 的 KL 散度）或“信心得分”。如果在某一步推理上，模型当前的 on-policy 策略已经能拿到很高的 Reward，且与专家的 token 完全不同，CHORD 会降低专家数据的权重，允许模型走出自己的解题路径；反之，如果模型在这里极其迷茫（价值网络预测极低），则拉高专家权重，强行拉回正轨

### HPT
HPT 采用的是另一种调度哲学：根据当前 batch 的在线表现来动态选择训练信号，在 exploitation demonstrations 和 stable exploration 之间切换。它同样说明，单阶段统一训练真正难的地方不是“把 loss 放在一起”，而是如何决定何时用哪种信号

## 优化层级与元学习关系的统一
上述大多还是在同一层级上混合两种信号：要么同一个目标里加权，要么同一个训练过程中切换。而这一层更进一步，认为 SFT 和 RL 的关系不应只是“并列混合”，而应该是 双层优化：一个层级负责探索和试错，另一个层级负责学习如何更好地指导这个探索过程。也就是说，这一层统一的是 两者的角色关系

### Beyond Two-Stage Training: Cooperative SFT and RL for LLM Reasoning (BRIDGE)
ICLR 2026 withdraw

BRIDGE 批评传统的 “SFT warm-up + RL fine-tuning” 两阶段范式过于解耦，导致 SFT 无法持续参与 RL 的优化过程。为此，它引入 bilevel optimization：下层执行 RL 更新，同时接收 SFT 监督；上层则显式优化 joint SFT-RL 相比纯 RL 的协同收益，也就是让 SFT 成为一个能够 meta-learn how to guide RL 的“领导者”。这和简单的混合 loss 非常不同，因为它不是问“比例是多少”，而是问“监督目标应如何塑造强化学习的搜索方向”

*   **内层（Inner Loop）**：模型在混合了 RL 和 SFT 信号的环境下更新参数（执行“试错+模仿”）
*   **外层（Outer Loop）**：计算内层更新后的参数在纯粹 RL 目标（或者某个验证集 Reward）上的表现。然后，梯度会**穿透内层的更新过程**，反向传播给 SFT 的控制参数（比如 SFT 样本的权重、或者 SFT 辅助 loss 的形式）

RL 负责试错，SFT 负责学会如何更好地组织这种试错，SFT 的存在是为了让最终的 RL 获得更高的 Reward
