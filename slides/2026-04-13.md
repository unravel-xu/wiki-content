# Unified Post-Training
打破 SFT 和 RL 的两阶段割裂，实现单阶段/统一后训练
如下我把 **Unified Post-Training** 拆成四个层次：沿着 **目标/梯度、数据、调度、优化层级** 这四个层次分类目前的工作

## 目标函数与梯度估计的统一
粗糙来说其核心思想：SFT 和 RL 表面上一个是监督学习、一个是强化学习，但从策略优化角度看，它们都在更新同一个语言策略，只是采用了不同的数据分布、不同的 advantage 构造方式、不同的稳定化手段。因此，“统一”首先可以发生在 **梯度估计器** 或 **目标函数分解** 上

### Towards a Unified View of Large Language Model Post-Training (HPT / UPGE)

它推导出一个 **Unified Policy Gradient Estimator, UPGE**，把大范围的后训练方法都写成同一目标下的不同梯度估计形式，并拆成四个可替换组件：**stabilization mask、reference policy denominator、advantage estimate、likelihood gradient**。它的关键贡献不是单独发明一个新 loss，而是说明很多后训练算法其实共享一套更底层的梯度骨架。基于这个统一视角，作者又提出 **HPT**，在训练中动态切换不同训练信号

### ORPO: Monolithic Preference Optimization without Reference Model

ORPO 可以看作这一层里更“工程化”的统一方式。它不再把“先 SFT、再 preference optimization”拆成两个阶段，而是直接把偏好优化项并到 SFT 的目标里，在 NLL 之外附加一个基于 **odds ratio** 的项，同时去掉 reference model。它没有像 HPT 那样给出一个总括性的统一梯度理论，但它清楚展示了：**alignment 目标本身可以被写进一个单阶段、单目标的训练式中**

## 思路一：从“底层梯度与优化器”视角进行统一

主要出发点：SFT 和 RL 看似目标不同，但在底层的梯度算子上是同源的

Towards a Unified View of Large Language Model Post-Training (HPT)
* 贡献：推导出了一个“统一策略梯度估计器 (UPGE)”，证明了几乎所有后训练方法（SFT, PPO, DPO）都可以拆解为四个通用组件（掩码、参考策略分母、优势估计、似然梯度）的乘积
* 工程实现：提出了 HPT (Hybrid Post-Training)。基于模型在当前 batch 的实时在线准确率表现进行“开关门”——准确率低就加大 SFT Loss 权重（手把手教），准确率高就加大 RL Loss 权重（鼓励自主探索）

ORPO: Monolithic Preference Optimization without Reference Model
* 贡献：这是较早提出“单阶段（Monolithic / Single-Stage）”概念的经典论文
* 工程实现：将偏好对齐（Alignment）直接融入 SFT 阶段。它不需要参考模型（Reference Model），也不需要单独的 RL 阶段，而是直接在传统的 SFT 交叉熵 Loss 后面加了一个基于赔率比（Odds Ratio）的惩罚项

## 思路二：从“动态调度与不确定性”视角进行统一

主要出发点：统一的关键在于“何时该用 SFT 兜底，何时该用 RL 探索”。除了 HPT 的按准确率分配，还有更细粒度的机制

SRFT: A Single-Stage Method with Supervised and Reinforcement Fine-Tuning for Reasoning
* 贡献：从“策略熵（Policy Entropy）”的视角揭示了 SFT 是引起全局粗粒度分布偏移，而 RL 是局部细粒度优化
* 工程实现：提出一种感知熵的加权机制（Entropy-aware weighting mechanism）。将 SFT 的演示数据和 RL 的在线探索数据放在同一个 Batch 里，利用模型的输出不确定性（熵）来动态分配两者的更新权重

SuperRL: Reinforcement Learning with Supervision to Boost Language Model Reasoning
* 贡献：针对 RL 在复杂推理任务中“奖励极度稀疏（Sparse Reward）”的痛点提出的自适应统一框架
* 工程实现：对于任何一个训练样本，如果在在线 Rollout 时模型生成的所有轨迹奖励全为 0（即完全找不到梯度方向），系统就会触发 **Fallback（回退）机制**，直接激活基于该问题专家数据的 SFT Loss

On-Policy RL Meets Off-Policy Experts: Harmonizing SFT and RL via Dynamic Weighting (Chord)
* 贡献：引入了“同策略（On-policy，即 RL）”与“异策略（Off-policy，即专家 SFT 数据）”的混合视角
* 工程实现：提出了 Chord 框架。SFT 不再是一个单独的微调阶段，而是作为 RL 探索过程中的“动态加权辅助 Loss”。它不仅有全局系数，还引入了 **Token 级别的细粒度权重分配**，防止模型对专家数据产生过拟合（Overfitting）

## 思路三：从“元学习与双层优化”视角进行统一
Beyond Two-Stage Training: Cooperative SFT and RL for LLM Reasoning (BRIDGE)
* 贡献：这是视角最新颖的一篇，用**双层优化**的数学框架来统一两者
* 工程实现：它将 SFT 视作“上层领导者（Upper-level leader）”，将 RL 视作“下层追随者（Lower-level follower）”。模型参数被拆分（例如 Base + LoRA），底层 RL 负责不断试错更新，而上层的 SFT 目标相当于一个元学习器（Meta-learner），用来指导 RL 的优化方向，最大化两者的协同增益
