### 思路1
把现在的 ACE 改成一个 **Adaptive ACE / Auto-ACE**
依据 task vector 的谱衰减程度，或者是条件数，CSP 强度
- 当 task covariance mismatch 大时，自适应 shrinkage 比固定 ridge 更稳定
- 当 tasks 接近同方差/同子空间时，方法退化到 averaging / task arithmetic / WUDI-like proxy
- merge error 与 heterogeneity、condition number、subspace overlap 的关系

从 task vector 恢复二阶结构 → 做 covariance-aware merging → 再把其中所有固定经验量都变成统计自适应量

### 思路2
不再逐层独立，而是对一个 block（如 attention QKV + proj，或 MLP up/down）联合建模，这样可把二阶代理从单矩阵扩展到 block covariance
从 layer-wise linear estimator 扩展为 block-structured / cross-layer estimator

### 思路3
提出一个基本的解释框架来分析所有的 data-free model merging 方法

## SFT & RL

### 能力分工 + 动态路由
- **高 SFT 分数并不可靠预测后续 RL 收益**  
    Quagmires 明确给出很多反例：SFT 更强的模型，后面 RL 不一定更强；更有预测力的是 held-out generalization loss 和 Pass@large k，而不是单纯 SFT 训练分数
- **SFT 与 RL 在目标层面不可完全解耦**  
    Non-decoupling 给出了更强的理论说法：若先把某一目标优化得很好，再切换到另一目标，后一阶段会系统性损伤前一阶段的最优性，不论顺序是 SFT→RL 还是 RL→SFT
- **SFT 与 RL 擅长的问题区间不同**  
    ReLIFT/Interleaved Online Fine-Tuning 的核心发现是：RL 更擅长在模型已有能力范围内提高表现，而 SFT 更擅长推动模型跨过“当前不会”的 hardest questions，给模型注入新的知识与推理模板
- **二者的优化几何也不同**  
    SRFT 认为 SFT 更像对策略分布做粗粒度、全局性的调整，而 RL 更像做细粒度、选择性的优化；“The Path Not Taken” 进一步指出 RLVR 更新偏向 off-principal directions，而 SFT 更偏 principal directions；RL Fine-Tuning Heals OOD Forgetting in SFT 则指出 SFT/RL 的作用差异与奇异向量旋转有关，而不仅是奇异值变化

> 后训练中不存在一个对所有样本、所有训练阶段、所有参数方向都最优的单一训练信号

样本复杂度分，对高难、长时程、低成功率样本，SFT 能显著降低达到首次有效学习信号的成本；对已有成功率基础的样本，RL 更能持续提高泛化与策略优化。SFT 与 RL 对同一参数不能普遍兼容，那么更合理的目标不是寻找全局固定训练顺序，而是**学习一个依赖于样本和训练状态的目标选择策略**，BRIDGE 这类 bilevel 工作，其实已经朝这个方向迈了一步：它不再把 SFT 仅仅当 warm-up，而是让 SFT 学会“怎样更好地服务 RL

后训练中的三相区间：不可探索区（SFT 区），边界区（Hybrid 区），可优化区（RL 区）
